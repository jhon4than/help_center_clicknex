Funções e Permissões

Gestor de permissões de cargos

O que são Funções?

É o gestor de permissões de cargos. Em vez de configurares cada utilizador um a um, crias "Funções" (ex: Vendedor, Suporte, Financeiro) com permissões específicas e atribuis essas funções aos teus operadores. Isto centraliza o controlo de acesso e facilita a gestão de equipas grandes.

Gestão Centralizada

Configure uma vez, aplique a muitos

Controlo Granular

Permissões específicas por módulo

Aplicação Instantânea

Mudanças refletem imediatamente

Como Utilizar

1. Criar Nova Função

Clica em "+ Nova Função" e configura:

  1. Nome da Função:
    • Define um nome claro (ex: "Gerente de Vendas")
    • Use nomes que reflitam o cargo real
    • Evite nomes genéricos como "Função 1"
  2. Cor de Identificação:
    • Escolhe uma cor para identificação rápida
    • Aparece em relatórios e listas
    • Facilita visualização no sistema
  3. Descrição (opcional):
    • Breve explicação do propósito da função
    • Ajuda outros administradores a entender

3. Selecionar Todas as Permissões

Se estiveres a criar uma função de administrador completo:

  • Usa o botão "Selecionar Todas" no topo
  • Marca todas as permissões de uma vez
  • Economiza tempo na configuração
  • Depois podes desmarcar permissões específicas se necessário

4. Gestão de Operadores por Função

Na tabela principal de funções, visualiza:

Coluna Operadores

  • Número de utilizadores vinculados a cada função
  • Clique no número para ver lista completa
  • Identifique funções mais usadas

Coluna Permissões

  • Total de permissões ativas naquela função
  • Comparação rápida entre funções
  • Identifique funções muito permissivas

Coluna Status

  • Padrão: Função aplicada automaticamente a novos operadores
  • Ativa: Função disponível para atribuição
  • Inativa: Função desabilitada (operadores mantêm acesso)

5. Ações Rápidas

No menu "..." de cada função:

Ver Operadores

  • Lista completa de quem possui aquele cargo
  • Acesso rápido aos perfis
  • Identifique quem será afetado por mudanças

Editar Permissões

  • Alterar permissões a qualquer momento
  • Aplicação Instantânea: Mudanças refletem imediatamente
  • Todos os operadores daquela função são atualizados
  • Não precisa relogar para aplicar

Duplicar Função

  • Cria cópia com as mesmas permissões
  • Útil para criar variações (ex: Vendedor Jr. e Vendedor Sr.)
  • Economiza tempo de configuração

Excluir Função

  • Remove a função do sistema
  • Operadores vinculados perdem todas as permissões
  • Requer confirmação de segurança
  • Não pode excluir função "Administrador" padrão

Dica de Segurança

A função "Administrador" é geralmente a Padrão e possui acesso total ao sistema.

  • Evita criar muitas funções com permissões de "Excluir"
  • Mantém a integridade dos teus dados
  • Apenas gestores de confiança devem ter acesso total
  • Revisa permissões regularmente
  • Use o princípio do "menor privilégio necessário"

Boas Práticas

  • Funções por Departamento: Crie funções que reflitam a estrutura da empresa (Vendas, Suporte, Financeiro)
  • Hierarquia Clara: Defina níveis (Jr., Pleno, Sr.) com permissões progressivas
  • Teste Antes: Crie um operador de teste para validar permissões
  • Documentação: Use o campo descrição para explicar o propósito de cada função
  • Revisão Trimestral: Reveja permissões a cada 3 meses
  • Auditoria: Use o módulo Sistema → Auditoria para ver quem fez o quê

Exemplos de Funções Comuns

Vendedor

  • Ver e criar Leads
  • Ver, criar e editar Negócios (apenas os seus)
  • Responder mensagens de WhatsApp
  • Ver relatórios de vendas
  • SEM permissão para excluir ou ver dados de outros vendedores

Atendente

  • Ver e responder conversas de WhatsApp
  • Criar agendamentos
  • Ver informações básicas de clientes
  • SEM acesso a vendas, financeiro ou configurações

Gerente

  • Ver todos os dados da equipe
  • Criar e editar em todos os módulos
  • Acesso a relatórios completos
  • Gerenciar operadores do seu setor
  • SEM acesso a configurações críticas do sistema

Variáveis

Gestor de campos dinâmicos do sistema

O que são Variáveis?

É o gestor de campos dinâmicos do sistema. As variáveis permitem inserir informações que mudam conforme o contexto (ex: nome do funcionário, nome da empresa, preço) dentro de mensagens padronizadas ou templates de atendimento. Isto personaliza a comunicação automaticamente, sem esforço manual do operador.

Como Utilizar

1. Criar Nova Variável

No formulário "Nova Variável", configura:

Nome da Tag

  • Define o nome começando sempre com @
  • Exemplos válidos:
    • @NOME_VARIAVEL
    • @FUNCIONARIO
    • @SITE
    • @PRECO_PRODUTO
  • Regras:
    • Sempre em MAIÚSCULAS
    • Usa underscore (_) para separar palavras
    • Sem espaços ou caracteres especiais
    • Deve ser descritivo e claro

2. Valor de Atribuição

Escolhe o tipo de dado que a variável representa:

Fixo (Estático)

  • O que é: Um texto que não muda
  • Quando usar: Informações constantes da empresa
  • Exemplos:
    • @SITE → www.teusite.com
    • @TELEFONE → +351 912 345 678
    • @EMAIL → contato@empresa.com
    • @HORARIO → Segunda a Sexta, 9h-18h
    • @ENDERECO → Rua Exemplo, 123
  • Preenche o campo "Valor Fixo" com o texto desejado

Variável (Dinâmico)

  • O que é: Dados que o sistema puxa automaticamente do banco de dados
  • Quando usar: Informações que mudam por contexto
  • Exemplos:
    • @FUNCIONARIO → Nome de quem está a atender
    • @CLIENTE → Nome do cliente na conversa
    • @LEAD_TELEFONE → Telefone do lead
    • @NEGOCIO_VALOR → Valor do negócio em aberto
    • @AGENDAMENTO_DATA → Data do próximo compromisso
  • Seleciona o campo do banco de dados correspondente

3. Uso nas Mensagens

Para usar uma variável, basta escrever a tag exatamente como a criaste no meio de qualquer texto:

Exemplo 1: Atendimento Personalizado

Mensagem com variável:

"Olá, sou o @FUNCIONARIO, como posso ajudar?"

Cliente recebe:

"Olá, sou o João Silva, como posso ajudar?"

Exemplo 2: Informações da Empresa

Mensagem com variável:

"Visite nosso site @SITE ou ligue para @TELEFONE"

Cliente recebe:

"Visite nosso site www.empresa.com ou ligue para +351 912 345 678"

Exemplo 3: Dados do Negócio

Mensagem com variável:

"Olá @CLIENTE, sua proposta de @NEGOCIO_VALOR está pronta!"

Cliente recebe:

"Olá Maria Santos, sua proposta de €1.500,00 está pronta!"

Onde Usar

  • Templates de WhatsApp: Mensagens padronizadas
  • Campanhas: Disparos em massa personalizados
  • Chatbots: Respostas automáticas
  • Agentes IA: Conversas inteligentes
  • Respostas Rápidas: Atalhos de texto

4. Gestão de Variáveis

Na lista à direita, podes ver e gerir todas as variáveis cadastradas:

Informações Exibidas

  • Tag: Nome da variável (ex: @FUNCIONARIO)
  • Tipo: Fixo ou Dinâmico
  • Valor: Conteúdo (se fixo) ou origem (se dinâmico)
  • Criado em: Data de criação
  • Usado em: Quantas mensagens/templates usam esta variável

Ações Disponíveis (Menu "...")

  • Editar: Alterar valor fixo ou origem dinâmica
  • Ver Uso: Lista onde a variável está sendo usada
  • Duplicar: Criar variação similar
  • Excluir: Remover variável
    • Cuidado: Mensagens que usam esta variável ficarão com texto quebrado
    • Sistema avisa se variável está em uso

Variáveis Dinâmicas Disponíveis

O sistema oferece acesso a diversos campos do banco de dados:

Dados do Operador

  • @FUNCIONARIO → Nome do operador
  • @FUNCIONARIO_EMAIL → Email do operador
  • @FUNCIONARIO_SETOR → Setor do operador

Dados do Cliente/Lead

  • @CLIENTE → Nome do contato
  • @CLIENTE_TELEFONE → Telefone
  • @CLIENTE_EMAIL → Email
  • @CLIENTE_EMPRESA → Nome da organização

Dados de Negócio

  • @NEGOCIO_TITULO → Nome do negócio
  • @NEGOCIO_VALOR → Valor monetário
  • @NEGOCIO_ETAPA → Etapa atual no pipeline

Dados de Agendamento

  • @AGENDAMENTO_DATA → Data do compromisso
  • @AGENDAMENTO_HORA → Horário
  • @AGENDAMENTO_SERVICO → Tipo de serviço

Dados da Empresa

  • @EMPRESA_NOME → Nome da empresa
  • @EMPRESA_SITE → Website
  • @EMPRESA_TELEFONE → Telefone principal

Dica: Personalização Aumenta Engajamento

O uso de variáveis dinâmicas aumenta a percepção de cuidado no atendimento, pois o sistema personaliza a conversa sem esforço manual do operador.

  • Clientes sentem-se mais valorizados
  • Taxa de resposta aumenta
  • Reduz erros de digitação
  • Economiza tempo da equipa

Exemplos Práticos de Uso

Boas-vindas Personalizado

"Olá @CLIENTE! Sou @FUNCIONARIO da @EMPRESA_NOME. Como posso ajudar hoje?"

Confirmação de Agendamento

"@CLIENTE, seu agendamento está confirmado para @AGENDAMENTO_DATA às @AGENDAMENTO_HORA. Nos vemos em breve!"

Follow-up de Proposta

"Olá @CLIENTE, gostaria de saber se teve oportunidade de analisar nossa proposta de @NEGOCIO_VALOR?"

Informações de Contato

"Para mais informações, visite @EMPRESA_SITE ou ligue para @EMPRESA_TELEFONE no horário @HORARIO"

Importante

  • Escreve tags exatamente como cadastradas (case-sensitive)
  • Não esqueças o @ no início
  • Se variável não existir, texto aparece literal para o cliente
  • Testa mensagens antes de usar em campanhas
  • Variáveis dinâmicas só funcionam se dados existirem no sistema

Boas Práticas

  • Nomes Descritivos: @PRECO_PRODUTO é melhor que @P1
  • Padronização: Mantém padrão de nomenclatura (MAIÚSCULAS, underscore)
  • Documentação: Usa descrição para explicar quando usar
  • Testes: Sempre testa variáveis antes de usar em produção
  • Fallback: Considera o que acontece se dado não existir
  • Revisão: Remove variáveis obsoletas periodicamente

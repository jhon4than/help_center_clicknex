Sistema / Auditoria

Auditoria e Logs

Registo histórico de todas as ações do sistema

O que é Auditoria?

É o registo histórico (logs) de todas as ações relevantes realizadas no sistema. Serve como ferramenta de segurança e transparência, permitindo rastrear alterações em leads, negócios, configurações ou acessos de utilizadores. É essencial para resolver dúvidas sobre "quem fez o quê" e garantir responsabilidade de cada operador.

Segurança

Rastreamento completo de ações

Transparência

Histórico imutável e auditável

Investigação

Resolva dúvidas rapidamente

Como Utilizar

1. Consulta de Histórico

No painel central, aparecem os logs detalhados de todas as ações:

Informações Exibidas

  • Usuário: Quem realizou a ação
    • Nome completo do operador
    • Email de login
    • Função/Cargo
  • Ação Realizada: O que foi feito
    • Criou, Editou, Excluiu, Visualizou
    • Login, Logout
    • Exportou dados
    • Alterou configuração
  • Entidade Afetada: Onde aconteceu
    • Lead, Negócio, Projeto, Agendamento
    • Nome ou ID do registro
    • Módulo do sistema
  • Data/Hora: Quando ocorreu
    • Timestamp preciso (até segundos)
    • Fuso horário do sistema
  • Detalhes: Informações adicionais
    • Valores antes e depois (em edições)
    • IP de origem
    • Dispositivo usado

2. Busca Avançada

Ferramentas poderosas para encontrar ações específicas:

Busca por Utilizador

  • Procura por nome ou email do operador
  • Ver todas as ações de uma pessoa específica
  • Útil para:
    • Auditar trabalho de um funcionário
    • Investigar suspeita de uso indevido
    • Avaliar produtividade

Busca por Entidade

  • Procura por nome de um lead ou ID de um negócio
  • Ver histórico completo de um registro
  • Exemplo: "João Silva" mostra:
    • Quem criou o lead
    • Quem editou dados
    • Quem criou negócio vinculado
    • Quem enviou mensagens

Busca por Texto Livre

  • Campo de busca geral
  • Procura em todos os campos simultaneamente
  • Útil quando não sabes exatamente o que procuras

3. Filtros Inteligentes

Utiliza os filtros de topo para segmentar a visualização:

Filtro por Tipo de Ação

  • Criação: Novos registros adicionados
  • Edição: Alterações em dados existentes
  • Exclusão: Registros removidos (crítico!)
  • Visualização: Acessos a dados sensíveis
  • Exportação: Downloads de dados
  • Login/Logout: Acessos ao sistema
  • Configuração: Mudanças em settings

Filtro por Módulo

  • Dashboard
  • Vendas (Leads, Negócios, Organizações)
  • Atendimento (WhatsApp, Campanhas)
  • Agenda
  • Projetos
  • Inteligência
  • Equipes
  • Sistema

Filtro por Período

  • Hoje: Últimas 24 horas
  • Esta Semana: Últimos 7 dias
  • Este Mês: Mês atual
  • Personalizado: Define data início e fim

Filtro por Severidade

  • Crítico: Exclusões, alterações de permissões
  • Alto: Edições importantes, exportações
  • Médio: Criações, visualizações
  • Baixo: Logins, navegação

4. Controlo e Responsabilidade

É a ferramenta ideal para resolver dúvidas e garantir accountability:

Casos de Uso Comuns

"Quem apagou este lead?"
  1. Busca pelo nome do lead
  2. Filtra por ação "Exclusão"
  3. Sistema mostra quem, quando e de onde apagou
"Quem alterou o valor deste negócio?"
  1. Busca pelo ID ou nome do negócio
  2. Filtra por ação "Edição"
  3. Ver detalhes mostra valor antes e depois
  4. Identifica quem fez a alteração
"Quem acessou dados sensíveis?"
  1. Filtra por módulo "Vendas" ou "Financeiro"
  2. Filtra por ação "Visualização" ou "Exportação"
  3. Ver lista de acessos com timestamps
"Quando este operador trabalhou?"
  1. Busca pelo nome do operador
  2. Filtra por período desejado
  3. Ver horários de login/logout e atividades

5. Detalhes de um Log

Clica em qualquer registro para ver informações completas:

Painel de Detalhes

  • Resumo da Ação: Descrição em linguagem natural
  • Dados Alterados: Comparação antes/depois (diff)
    • Campos modificados destacados
    • Valores antigos em vermelho
    • Valores novos em verde
  • Contexto Técnico:
    • IP de origem
    • User Agent (navegador/dispositivo)
    • Sessão ID
    • Request ID (para suporte técnico)
  • Ações Relacionadas: Outras ações do mesmo usuário no mesmo período

6. Exportação de Logs

Para análises externas ou conformidade legal:

Como Exportar

  1. Aplica filtros desejados
  2. Clica em "Exportar Logs"
  3. Escolhe formato:
    • CSV: Para Excel/análise
    • JSON: Para processamento técnico
    • PDF: Para documentação/impressão
  4. Download inicia automaticamente

Quando Exportar

  • Auditorias internas trimestrais
  • Investigações de segurança
  • Conformidade LGPD/GDPR
  • Processos legais
  • Backup de registros importantes

Destaque: Imutabilidade

Os logs de auditoria são gerados automaticamente pelo sistema e não podem ser editados pelos operadores.

  • Garante autenticidade das informações
  • Nem administradores podem alterar logs
  • Timestamp criptograficamente assinado
  • Conformidade com normas de auditoria
  • Prova legal em disputas

Ações Registradas Automaticamente

Módulo Vendas

  • Criação, edição, exclusão de Leads
  • Criação, edição, exclusão de Negócios
  • Mudança de etapa no pipeline
  • Alteração de valores
  • Atribuição a operadores

Módulo Atendimento

  • Envio de campanhas
  • Criação de templates
  • Transferências de conversas
  • Encerramento de atendimentos

Módulo Equipes

  • Criação de operadores
  • Alteração de permissões
  • Mudança de funções
  • Desativação de usuários

Módulo Sistema

  • Alterações em configurações
  • Conexão/desconexão de WhatsApp
  • Criação de tabulações
  • Modificação de variáveis

Segurança

  • Login bem-sucedido
  • Tentativas de login falhadas
  • Logout (manual ou automático)
  • Alteração de senha
  • Exportação de dados

Boas Práticas

  • Revisão Regular: Verifica logs semanalmente para identificar anomalias
  • Investigação Rápida: Usa filtros para encontrar informações rapidamente
  • Backup: Exporta logs importantes mensalmente
  • Treinamento: Ensina equipa que ações são rastreadas
  • Transparência: Informa operadores sobre auditoria (reduz má conduta)
  • Conformidade: Mantém logs por período legal exigido (geralmente 5 anos)

Retenção de Dados

  • Logs são mantidos por 90 dias no sistema ativo
  • Após 90 dias, são arquivados automaticamente
  • Logs arquivados podem ser recuperados mediante solicitação
  • Exporta logs importantes antes do arquivamento
  • Conformidade LGPD: Dados podem ser solicitados por titulares
